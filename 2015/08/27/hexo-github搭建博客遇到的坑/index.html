hexo+github搭建博客遇到的坑

发表于 2015-08-27   |     |   阅读次数

hexo官网

环境：

  1. OSX 10.10.4
  2. Homebrew 0.9.5
  3. Node v0.12.7
  4. npm 2.12.1
  5. hexo 3.1.1

坑

一、安装Homebrew

二、安装Node.js

它将在你的机器上安装 Node.js 和 npm (node package manager).

三、安装hexo

-g表示全局

四、生成blog

不加–no-optional在我这里出现：

  • blog
    • _config_yml // 注配置文件
    • db.json // 数据
    • debug.log // 调试日志
    • _node_mudules // nodejs 相关依赖
    • package.json // 配置依赖
    • scaffolds // 脚手架 - 也就是一个工具模板
    • source // 存放blog正文的地方
    • themes // 存放皮肤的地方

启动服务

在浏览器中输入：localhost:4000 即可访问

五、部署到github

在github上建立一个Repository

如果自己有域名的可以不这么建，没有的话就这样建，不然有些css读不出来

修改 blog 目录下的 _config.yml 文件

执行命令，先生成静态界面，然后部署到github上

会要你输入github用户名和密码

你会发现public 目录下的页面已经发布到github gh-pages 分支了
浏览器中输入username.github.io就可以访问到你的博客了

六、新建文章

当我们想写一篇blog时 在blog根目录下我们可以通过 hexo new <title> 指令来实现
hexo会在 source/_posts/ 下新建title.md 文件。
编辑title.md就是编辑文章了
使用markdown语法
hello.md 的文档和目录可以这样添加

七、使用主题

next主题

UML插件

安装遇到的问题
npm install hexo-diagram –save

由于之前装得时候是用的 sudo 所以出现以上错误

这个时候就是node-gyp的问题了

http://stackoverflow.com/questions/17967546/common-gypi-not-found-error-in-node-js

然后再这样就安装好了
