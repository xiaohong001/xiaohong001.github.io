Git权威指南

git的一些命令

  1. git config 设置
  2. git init 初始化
  3. git add 添加文件
  4. git commit -m “msg” 提交
  5. git diff filename 对比不同
  6. git status 查看仓库状态
  7. git log (–pretty=oneline) 查看提交记录，参数是一行显示
  8. git reset –hard HEAD^ 返回上一个版本
  9. git reflog 查看每一次使用的历史命令
  10. git checkout – file可以丢弃工作区的修改，总之，就是让这个文件回到最近一次git commit或git add时的状态。
  11. git reset HEAD file 可以把暂存区的修改撤销掉
  12. git rm file 在本地删除的也删除
  13. git remote add origin git@server-name:path/repo-name.git 关联远程仓库
  14. git push -u origin master 添加并且关联远程仓库 -u 只在第一次关联的时候需要
  15. git clone 从远程仓库克隆下来
  16. git checkout -b dev 表示将创建并切换到dev分支 等价于后面两条命令
  17. git branch dev 创建分支
  18. git checkout dev 切换分支
  19. git branch 查看当前分支
  20. git checkout master 切换到master分支
  21. git merge dev 合并dev分支
  22. git branch -d dev 删除dev分支
  23. git log –graph –pretty=oneline –abbrev-commit 查看分支合并图
  24. git merge –no-ff -m “merge with no-ff” dev 合并分支时禁用Fast forward，并生成一条新的commit
  25. git stash 保存工作现场
  26. git stash pop 恢复工作现场
  27. git branch -D 强行删除没有合并的分支
  28. git remote (-v) 查看远程查库(显示详细信息)
  29. git push origin branch-name 从本地推送分支
  30. git checkout -b branch-name origin/branch-name 在本地创建和远程分支对应的分支
  31. git branch –set-upstream branch-name origin/branch-name 建立本地和远程分支的关联
  32. git tag 创建tag，默认打在HEAD上
  33. git tag -a -m “blablabla…”可以指定标签信息；
  34. git tag -s -m “blablabla…”可以用PGP签名标签；
  35. git tag可以查看所有标签。
  36. git show
  37. git push origin 可以推送一个本地标签；
  38. git push origin –tags可以推送全部未推送过的本地标签；
  39. git tag -d 可以删除一个本地标签；
  40. git push origin :refs/tags/可以删除一个远程标签。
  41. git blame 查看文件修改记录
  42. git diff 查看文件的不同 @@ -1,7 +1,7 @@ 表示从第一行开始，到第七行的代码
  43. git pull 就是 git fetch 和 git merge 组成的。
  44. git cherry-pick 应用某一个提交的修改
  45. git rebase master feature 一个分支的所有修改在另一个分支上重新应用一遍
  46. git repack 将版本库未打包的松散对象打包
  47. git grep TODO 搜索TODO
  48. git rebase -i hash 重命名提交
  49. git add -p 提交部分
  50. git submodule add 添加远程依赖模块
  51. git rm -r –cached . 删除已经add，但还没commit的文件
  52. git ls-files 查看所以版本控制的文件

注意点

git 中 HEAD表示当前版本,上一个版本就是HEAD^，上上一个版本就是HEAD^^
Git用<<<<<<<，=======，>>>>>>>标记出不同分支的内容
master分支上的是稳定版本，工作时另开一个dev分支，修改bug时，先保存当前工作现场，然后从需要的分支上创建个分支，等修改完后提交，切换到工作分支，恢复工作现场

git config –global alias.lg “log –color –graph –pretty=format:’%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset’ –abbrev-commit”

与 reset 不同的是，revert 只会撤销当前的 commit，而之后的 commit 操作的修改还会保留，但是reset 还会将之后的所有 commit 操作的修改全部退回 staging area 或丢弃。
