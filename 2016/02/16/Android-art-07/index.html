第七章

Android动画深入分析
Android的动画可以分为三种：View动画，帧动画，属性动画，其实帧动画也属于View动画的一种，只不过它和平移，旋转等常见的View动画表现形式上略有不同而已。

View 动画

View动画的作用对象是View，它支持4种动画效果，分别是平移动画，旋转动画，缩放动画和透明度动画。

View 动画的种类

View动画的四种变换效果对应着Animation的四个子类：TranslateAnimation、ScaleAnimation、RotateAnimation和AlphaAnimaition。可以通过代码和xml创建，推荐使用xml创建。

名称 标签 子类 效果
平移动画 <translate> TranslateAnimation 移动View
缩放动画 <scale> ScaleAnimation 放大或缩小View
旋转动画 <rotate> RotateAnimation 旋转View
透明度动画 <alpha> AlphaAnimation 改变View的透明度

View动画既可以是单个动画，也可以是多个动画的集合。
<set>标签对应AnimationSet类，它可以包含若干个动画。

  1. android:interpolator
    表示动画集合所采用的插值器，插值器影响动画的速度，这个属性可以不指定，默认为@android:anim/accelerate_decelerate_interpolator，即加速减速插值器。
  2. android:shareInterpolator
    表示集合中的动画是否和集合采用同一个插值器。
  3. <translate>
    平移动画，对应TranslateAnimation类，它可以使一个View在水平和竖直方向完成平移的动画效果。
    1. android:fromXDelta 表示x的起始位置
    2. android:toXDelta 表示x的结束位置
    3. android:fromYDelta 表示y的起始位置
    4. android:toYDelta 表示y的结束位置
  4. <scale>
    缩放动画，对应ScaleAnimation类，它可以使一个View具有放大和缩小的动画效果。
    1. android:fromXScale 水平方向缩放的起始值
    2. android:fromYScale 竖直方向缩放的起始值
    3. android:toXScale 水平方向缩放的结束值
    4. android:toYScale 竖直方向缩放的结束值
    5. android:pivotX 缩放的轴点的x坐标
    6. android:pivotY 缩放的轴点的y坐标
  5. <rotate>
    旋转动画，对应RotateAnimation类，它可以使一个View具有旋转的动画效果。
    1. android:fromDegrees 旋转开始的角度
    2. android:toDegrees 旋转结束的角度
    3. android:pivotX 旋转中心点的x坐标
    4. android:pivotY 旋转中心点的Y坐标
  6. <alpha>
    透明度动画，对应AlphaAnimation类，它可以改变View的透明度。
    1. android:fromAlpha 透明度的起始值
    2. android:toAlpha 透明度的结束值
  7. 常用属性
    1. android:duration 动画持续时间
    2. android:fillAfter 动画结束后View是否停留在结束位置。

自定义View动画

除了系统提供的四种View动画外，我们还可以自定义View动画。
派生一种新动画只需要继承Animation抽象类，然后重写它的initialize和applyTransFormation方法，在initialize方法中做一些初始化操作，在applyTransFormation方法中进行响应的矩阵变换即可，很多时候需要采用Camera来简化矩阵变换的过程。

属性动画

属性动画是api 11加入的新特性，和View动画不同，它对作用对象进行了扩展，属性动画可以对任何对象做动画，甚至还可以没有对象。属性动画中有ValueAnimator，ObjectAnimator和AnimatorSet等概念，通过它们可以实现绚丽的动画。

使用属性动画

属性动画可以对任意的对象属性进行动画而不仅仅是View，动画默认时间间隔是300ms，默认帧率为10ms/帧。其可以达到的效果是：在一个时间段内，完成对象从一个属性值到另一个属性值的改变。只要对象有这个属性，它都能实现动画效果。属性动画api 11以上才有，nineoldandroids开源库可以兼容之前的版本。
但是nineoldandroids 在api 11之前的版本都是使用代理的View动画来实现的，本质还是View动画。
常用的几个动画类：ValueAnimator，ObjectAnimator和AnimatorSet；

  1. 改变一个对象的translationY属性

  2. 改变一个对象的背景色属性，3秒内改变一次颜色，无限循环改变颜色

  3. 动画集合，5秒内水平移动，水平缩放，水平翻转

也可以用xml来定义，在xml中，<set>标签对应AnimatorSet，<animator>标签对应ValueAnimator，<objectAnimator>标签对应ObjectAnimator。
<set>标签的android:ordering有两个值可选together（所有动画同时播放）和sequentially（所有动画顺序播放），默认是together
<objectAnimator>和<animator>标签类似，<animator>只是少了一个android:propertyName属性，其他一样。

  1. android:propertyName
    表示属性动画的作用对象的属性名称
  2. android:duration
    动画的时长
  3. android:valueFrom
    属性的起始值
  4. android:valueTo
    属性的结束值
  5. android:startOffset
    表示动画的延迟时间，延迟多久执行
  6. android:repeatCount
    表示动画重复次数，默认是0，-1表示无限循环
  7. android:repeatMode
    动画重复模式，有restart，reverse可选，前者是重新播放，后者是第一次顺序播放，第二次逆序播放，第三次又顺序播放。
  8. android:valueType
    表示android:propertyName的取值类型，有floatType，intType，colorType和pathType可选

理解插值器和估值器

TimeInterpolator中文翻译为时间插值器，它的作用是根据时间流逝的百分比来计算当前属性值的百分比，系统内置有LinearInterpolator（线性插值器：匀速动画），AccelerateDecelerateInterpolator（加速减速插值器：两头慢中间快），DecelerateInterpolator（减速插值器：动画越来越慢）等。TypeEvaluator的中文翻译为类型估值算法，即估值器，它的作用是根据当前属性的百分比来计算改变后的属性值，系统预置的有IntEvaluator（针对整形属性），FloatEvaluator（针对浮点属性）和 ArgbEvaluator（针对color属性）
插值器和估值算法除了系统提供的外，我们还可以自定义。插值器和估值算法都是一个接口，且内部都只有一个方法，我们只要派生一个类实现接口，然后就可以做出千奇百怪的动画效果了。具体一点就是：自定义插值器需要实现Interpolator或者TimeInterpolator，自定义估值算法需要实现TypeEvaluator。另外对于其他的 属性类型做动画，就必须要自定义估值算法了。

属性动画的监听器

属性动画提供了监听器用于监听动画的播放过程，主要有如下两个接口：AnimatorUpdateListener和AnimatorListener

系统提供了AnimatorListenerAdapter类来选择需要实现的方法，而不是实现AnimatorListener的全部方法。

AnimatorUpdateListener比较特殊，它会监听整个动画过程，动画是由许多帧组成的，每播放一帧，onAnimationUpdate就会被调用，我们可以用这个特性，做一些特殊的事情。

对任意属性做动画

属性动画要求动画作用的对象提供该属性的get和set方法，属性动画根据外界传递的该属性的初始值和最终值，以动画的效果去调用set方法，每次传递给set的值都不一样，确切的来说是随着时间的推移，所传递的值越来越接近最终值。
所以，我们对Object的属性abc做动画，动画生效需要满足的条件：

  1. object必须提供setAbc方法，如果动画的时候没有传递初始值，还需要提供getAbc方法，因为系统要去取初始值（不满足时，程序Crash）。
  2. object的setAbc对属性abc所做的改变必须能通过某种方法表现出来 （不满足时，动画无效）。

不满足属性动画的要求时，官方给出来3中解决办法：

  1. 给你的对象属性加上get和set方法，如果你有权限的话

  2. 用一个类包装原始对象，间接为其提供get和set方法

  3. 采用ValueAnimator，监听动画过程，自己实现属性的改变

属性动画的工作原理

属性动画要求动画作用的对象提供该属性的get和set方法，属性动画根据外界传递的该属性的初始值和最终值，以动画的效果去调用set方法，每次传递给set的值都不一样，确切的来说是随着时间的推移，所传递的值越来越接近最终值。

使用动画的注意事项

  1. OOM问题
    主要出现在帧动画中，尽量避免使用帧动画
  2. 内存泄漏
    在属性动画中有无限循环的动画时，activity退出时应该及时把动画停止，否则会造成Activity无法释放而导致内存泄漏。View动画不存在此问题。
  3. 兼容性问题
    动画在3.0版本以下有兼容性问题
  4. View动画的问题
    View动画是对View的影像做动画，并不是真正的改变View的状态，因此有时候会出现动画完成后无法隐藏的现象。这时候要用view。clearAnimation()清除View动画即可解决。
  5. 不要使用px
    尽量使用dp，使用px会导致在不同设备上有不同效果
  6. 动画元素的交互
    将View移动后，在3.0以前的系统上，不管是View动画还是属性动画，新位置均无法触发点击事件，同时，老位置依然可以点击。3.0之后的View动画单击事件依然在原位置。
  7. 硬件加速
    使用动画的时候，建议开启硬件加速，提供动画的流畅性。
